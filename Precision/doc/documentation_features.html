Basic compiling instructions:

  • With the source files:
    • Include the Precision and Src directories in the compiler's search list, so that the translation units can see the proper headers and that the cpp files in Src may be compiled.
  • Just the headers:
    • Within your project files, include the proper headers. There is no extra step needed.
  • Compile.
  • If you don't wish to compile the source files, you may request a compiled static library to be sent to you by e-mail.


Current features:

  • Multi-precision number types ( Precision::Int, Precision::Float, Precision::Fract and their unsigned counterparts)
    • Basic arithmetic operations
    • Type aliases to ease use of number type components
    • Ability to convert to a string of images (std::basic_string<CharT>)
      • The visual representation may be in regular or scientific format.
    • Various tests such as even() and positive()
    • Shift functions to change the order of magnitude
    • Methods to retrieve individual digits
    • Int
      • Bitwise logical operations
      • Base dependent logical operations
    • Float
      • All comparators are based upon precision (e.g. FloatObj1 == FloatObj2 is safe to use)
      • Count digits on the left and/or right side of the decimal point
      • Keeps track of precision (immutable after construction unless the object is replaced via assignment)
    • Fract
      • Retrieve the numerator, denominator, and the decimal form
      • Memory of precision for the decimal form
      • Ability to cast to Float
    • Float and Fract
      • Tests if the number is an integer
      • Member methods for exponentiating and inverting
      • Ability to cast to Int
  • General number base templates ( Precision::General_Base::Int, Precision::General_Base::Float, Precision::General_Base::Fract and their unsigned counterparts)
    • Float for bases other than 10 has not been extensively tested.
    • For more details on how to instantiate a new number base, click here.
  • Dynamic number base templates ( Precision::Dynamic_Base::Int)
    • These types determine their bases during runtime, meaning their bases can also be changed during runtime.
    • The iterator type to access digit images is also generalized, so the client is not restricted to CharT const *const. Like the base, the image set may also be switched out during runtime.
  • Extra number bases ( Precision::Hexadecimal, Precision::Octal, Precision::ABC)
  • A set of precision make functions to generate numbers types from binary to base 36. For more information, click here.
  • String literal operators for default instantiations (e.g. "1E7"_p_int)
  • Psuedo random number generator ( Precision::Random)
  • Exception system
  • A limited mathematics system
  • A tag system to determine which category a number type falls under


Features currently under construction or are being repaired:

  • A stable exponentiation algorithm for Precision::General_Base::Fract.
  • Complex Number System ( Precision::Complex)
  • Number Sequence System ( Precision::Sequence, Precision::StrictSequence, Precision::RecursiveSequence, Precision::StrictRecursiveSequence)
  • Mathematics System (Precision::Math)
    • Functions under Precision/General_Base/Impl/Math_Shared_Functions.h (included by Precision/Precision_Math.h) are fully functional
  • Static Base ( Precision::Static)
    • Moving the old general base implementation into the Static name space. Type aliases will be provided, so that the types can still be accessed outside the Static name space.
  • Dynamic Base ( Precision::Dynamic)
    • Extend the Dynamic Base Number Type system to include floating point and fractional types.
    • Move the core of the implementation to an abstract type.
    • Making the Dynamic Base Number types the core of the Static Base Number types.
  • Abstract ( Precision::General_Base::Abstract)
    • Extend the Abstract Number Type system to include floating point and fractional types.


Future Features:

  • A revised system to replace the ugly constants in Precision/General_Base/Impl/Shared_Constants.h with compile-time constants
    • Possibly have constants system support general base as well (either by conversion from base 10 or by metaprogramming)
  • New algorithms to aid in converting number bases for floating point, fraction, and complex number systems
  • Extending the psuedo random number generator to also generate Float and Fract numbers
    • Perhaps also the Complex


Features Under Consideration or Research:

  • Multi-dimensional matrices system
  • A faster algorithm for converting among number bases
  • Making the main number types: Int, Float, and Fract; STL-like with iterators
  • A new system to aid in passing container types with optional template parameters for the passed container types
    • That is: template <template <typename...> class Container> will have a way to pass in additional arguments to modify the container type, such as passing custom allocators
  • Conservative number type system.
