Number Type System

This system is the heart of the library, providing number types of arbitrary precision and the ability to instantiate custom number bases.

[ Skip to the list of related classes and functions.]

A default instantiation of base 10 numbers is provided. The hexadecimal base, octal base, and an exotic base are also provided.

The number types currently provided by the Number Type System are:

  • Int
  • Float
  • Fract
  • UInt
  • UFloat
  • UFract
Number types still under construction include:
  • Complex
The namespace qualifiers of each provided instantiation are:
Base 10
Precision::FOO
Hexadecimal
Precision::Hexadecimal::FOO
Octal
Precision::Octal::FOO
ABC
Precision::ABC::FOO
So, for example, to use the Float class in the Octal base, one would use Precision::Octal::Float.


The Number Type System also contains templates for instantiating number types of custom bases. For instantiating these types, the following information is required (in the same order one would pass them as template arguments):
  1. typename CharT - The type of character or image used to represent each digit in Base N. Must be compatible as a template parameter to std::basic_string<T>. The string type (str_type) of the class shall be instantiated as std::basic_string<CharT>. The following or the equivalent of the following functions must be supported for type CharT:
    • std::istream& operator>>(std::istream&, CharT&);
    • bool operator==(const CharT&, const CharT&);
  2. CharT const* _0 - A pointer to the first digit in an array containing the images of each digit. The array must have external linkage. Important note: It is the responsibility of the instantiator to ensure _0 is a valid parameter and that the array pointed to contains the appropriate images.
  3. typename ByteType - The type used for the computer representation of each digit. This type also sets the maximum base that may be used and affects the dynamic storage size. The library provides a default type: Precision::byte_type, which is sufficient for bases 127 and below. This default type is also the default value of the argument.
    Note: The bahaviour of the instantiated types is not guaranteed for bases below 2.
  4. ByteType N - The base N the class shall represent. The default value of this argument is 10.
More information on how to instantiate the number types can be found here.
