Decimal Literal Operators

Associated File: Precision/General_Base/Precision_Literal_Operators.h

For the decimal instantiation, there is a set of literal operators in the global space. They allow the client to construct objects with the usual parameters, i.e. integer values and string objects. For each supported number type, there are two literal operators that accept strings. One parses the string as if calling the constructor while the other parses the string as if it contained an order of magnitude, e.g. "123.45 E 67", which is equivalent to writing "123.45 * 1067".

Precision::Int
Int operator"" _P_Int(char const *const, std::size_t)
Construct a Precision::Int object from a string. E.g. "123"_P_Int

Int operator"" _P_Int_E(char const *const, std::size_t)
Construct a Precision::Int object from a string where the number is followed by an order of magnitude. E.g. "142.22E-2"_P_Int_E

Int operator"" _P_Int(unsigned long long)
Construct a Precision::Int object from an unsigned integer value. E.g. 237864_P_Int
Note: Remember that, even if the integer is prefixed with a negative sign, the sign is parsed as a lower precedent operator, i.e. writing -123_P_Int is equivalent to -(123_P_Int).
