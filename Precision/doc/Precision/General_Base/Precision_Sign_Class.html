Precision::SignClass

A class used by the number types in the library to denote whether or not they are positive or negative. This is a relatively lightweight class acting as a proxy such that the client need not worry about the details.

Associated File: Precision/General_Base/Precision_Sign_Class.h

Modifiers

Reverse the sign, e.g. if the class value is 1, it becomes -1 and vice versa. Both flip() and negate() do the same thing.
void assign(signed_type new_value)
Replace the current value of the class with a new value. If new_value is 0 or positive, the new value is 1 (positive). Otherwise, it is -1 (negative).
void make_positive()
Replace the current value of the class with 1 (positive). This is as if calling assign(1).
void make_negative()
Replace the current value of the class with -1 (negative). This is as if calling assign(-1).

Example Use
  1. #include "Precision/General_Base/Precision_Sign_Class.h"
  3. #include <iostream>
  5. int main(){
  6.     Precision::SignClass sign(-1); //Negative
  7.     std::cout << std::boolalpha;
  8.     std::cout << "Value: " << sign.value() << '\n';

  10.     std::cout << "Negative? " << sign.negative() << '\n';
  12.     int val = sign; //Implicit conversion
  13.     sign.negate(); //sign is now positive
  14.     sign.assign(val); //sign is now negative
  15.     sign.make_positive(); //Value is positive
  16.     sign.make_negative(); //Back to negative
  18.     return 0;
